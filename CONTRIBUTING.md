# Contributing to Namma-Kathey

First off, thank you for considering contributing to Namma-Kathey! It's people like you that make the project better.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue and include:
* A clear title and description.
* Steps to reproduce the bug.
* Expected behavior vs actual behavior.
* Screenshots if applicable.

### Suggesting Enhancements
We welcome ideas to make the heritage explorer more engaging!
* Open an issue with the tag "enhancement".
* Describe the feature and why it would be useful for users.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the app builds and runs without errors.
4. Use clear, descriptive commit messages.

## Style Guide
* Follow standard Kotlin coding conventions.
* Keep XML layouts optimized and use meaningful IDs.
* Ensure all strings are internationalized in `strings.xml` (both EN and KN).

## New Content
If you want to add a new Hero or District:
1. Update `DistrictDataSource.kt` or `HeroDataSource.kt`.
2. Add the corresponding strings in `strings.xml`.
3. Provide high-quality images in the `res/drawable` folder.

Thank you for helping us preserve and promote Karnataka's heritage!
